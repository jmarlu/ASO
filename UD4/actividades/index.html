    🔧 Actividades – ACL en sistemas de ficheros (Integración de servicios)

    Nivel 1 – Fundamentos

    1. Comprueba soporte ACL
    2. Muestra con mount o tune2fs si tu FS tiene acl.

    3. Si no aparece, remonta con acl y captura el antes/después.

    4. Lectura de ACL

    5. Crea ~/demo-acl y dos archivos.
    6. Ejecuta getfacl y explica qué significa la mask.

    Nivel 2 – Aplicación

    1. Proyecto compartido
    2. Crea /srv/proyecto con dueño root:proyecto y chmod 2770.
    3. ACL: grupo equipo rwX, usuario ana rX, usuario luis rwX, defaults coherentes.

    4. Valida con touch como cada usuario (usa sudo -u) y demuestra herencia.

    5. Limpieza y reversión

    6. Quita todas las ACL de /srv/proyecto con setfacl -b y muestra cómo vuelve al modo POSIX clásico.

    Nivel 3 – Integración con servicios

    1. Samba o NFS
    2. Exporta /srv/proyecto por Samba o NFS.

    3. Muestra que desde el cliente se respetan las ACL (intenta escribir con un usuario permitido y uno denegado).

    4. Auditoría rápida

    5. Escribe un script que liste directorios con + en ls -l bajo /srv y saque getfacl de cada uno a un log con sello de fecha.

    Nivel 4 – Reto opcional

    1. Reto Ansible (ACL de red)
    2. Escribe un playbook que configure una ACL en routers/switches Cisco (o emulados) usando un template y when por host_vars.
    3. Incluye tarea de validación (por ejemplo, ios_command que muestre la ACL aplicada).
    4. Entrega inventario, playbook y salida de ejecución.

    🧪 Laboratorio integrado (LDAP Docker + Fileserver LXD con ACL, NFS/Samba)

    Objetivo: identidad central en LDAP (Docker, con TLS), servidor de ficheros en LXD que resuelve usuarios LDAP, crea homes al vuelo y expone una carpeta compartida con ACL, accesible por NFS y/o Samba.

    1. Preparar fileserver en LXD
    2. Crea contenedor Ubuntu con security.nesting=true.
    3. Instala sssd-ldap libpam-sss libnss-sss ldap-utils acl.
    4. Configura /etc/sssd/sssd.conf con tu ldap_uri = ldaps://<IP LDAP>:636, ldap_tls_cacert = /etc/ssl/certs/ca-ldap.crt, ldap_search_base = dc=....
    5. chmod 600 /etc/sssd/sssd.conf, systemctl enable --now sssd.
    6. PAM: añade session required pam_mkhomedir.so skel=/etc/skel umask=077 a common-session.

    7. Verifica: getent passwd alumno1 devuelve UID/GID LDAP.

    8. Homes y carpeta compartida con ACL

    9. Crea base de homes /home/usuarios (755).
    10. Crea /srv/compartida con chown root:equipo, chmod 2770, default ACL setfacl -m d:g:equipo:rwX /srv/compartida y ACL explícita setfacl -m g:equipo:rwX /srv/compartida.

    11. Prueba: su - alumno1 crea /home/usuarios/alumno1 700; sudo -u alumno1 touch /srv/compartida/ok funciona; usuario fuera de equipo falla.

    12. Exporta por NFS

    13. /etc/exports ejemplo:
      /srv/compartida 10.0.0.0/24(rw,sync,subtree_check,acl,root_squash,fsid=0)
      /home/usuarios 10.0.0.0/24(rw,sync,subtree_check,acl,root_squash)
    14. Reinicia nfs-kernel-server.

    15. Cliente: mount -t nfs filesrv:/srv/compartida /mnt/comp y prueba escritura/denegación según ACL.

    16. Exporta por Samba (opcional o alternativo)

    17. Instala samba; añade a smb.conf:
      [global] security = user; vfs objects = acl_xattr; inherit permissions = yes
      [compartida] path = /srv/compartida; read only = no; create mask = 0660; directory mask = 2770; valid users = @equipo
      [homes] browseable = no; read only = no; create mask = 0600; directory mask = 0700
    18. testparm y systemctl restart smbd.

    19. Cliente: smbclient //filesrv/compartida -U alumno1 (ok) y prueba con usuario fuera de grupo (denegado).

    20. Validaciones finales

    21. openssl s_client -connect <IP LDAP>:636 -CAfile ca-ldap.crt (TLS ok).
    22. getent passwd/group de varios usuarios/grupos LDAP.
    23. ls -ld /srv/compartida → drwxrws---+; getfacl /srv/compartida muestra defaults.

    24. Pruebas de acceso desde cliente NFS/Samba según pertenencia a equipo.

    25. Entrega

    26. Capturas/comandos de: configuración sssd.conf (sin contraseñas), getent, creación de home, getfacl de /srv/compartida, prueba ok/fallo en escritura, montaje NFS o acceso SMB, y comprobación de TLS LDAP.
